## 1. Frontend Deployment: Vercel (100% Free)

Vercel is the premier platform for deploying Vite/React applications. Its free **Hobby Tier** is exceptionally generous and fully sufficient for DevPulse.

### Step-by-Step Vercel Setup
1. **Prepare your Frontend config:**
   Make sure `frontend/vite.config.ts` handles client routing properly. (Since we are using Vite, Vercel needs a `vercel.json` in the root of the `frontend/` directory to redirect all routing requests to `index.html`).
   Create `frontend/vercel.json`:
   ```json
   {
     "rewrites": [
       { "source": "/(.*)", "destination": "/index.html" }
     ]
   }
   ```

2. **Connect to Vercel:**
   - Log into [Vercel](https://vercel.com) using your GitHub account.
   - Click **Add New** → **Project**.
   - Import the `devpulse` repository.

3. **Configure Build Settings:**
   - **Framework Preset:** Vite
   - **Root Directory:** `frontend`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`

4. **Add Environment Variables:**
   Add these in Vercel’s project settings:
   - `VITE_SUPABASE_URL` = `https://your-project.supabase.co`
   - `VITE_SUPABASE_PUBLISHABLE_KEY` = `your-supabase-anon-key`
   - `VITE_API_URL` = `https://your-backend-url.onrender.com` (Use the Render url you generate below)

5. **Deploy:** Click **Deploy**. Vercel will build the frontend and provide you with a production-ready, SSL-secured domain (e.g., `devpulse-frontend.vercel.app`).

## 2. Backend Deployment: Render (Student & Free Friendly)

Render offers a very simple platform for deploying Node.js/Express applications. 

### Why Render is Great for Students:
- Offers a **Free Web Service** tier.
- With the **GitHub Student Developer Pack**, Render provides **free credits** to unlock paid services (which prevents server sleep/spin-down).

### Step-by-Step Render Setup (Free Web Service)
1. Log into [Render](https://render.com) using your GitHub account.
2. Click **New** → **Web Service**.
3. Link your `devpulse` repository.
4. **Configure Settings:**
   - **Name:** `devpulse-backend`
   - **Language:** `Node`
   - **Root Directory:** `backend`
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `node dist/server.js`
   - **Instance Type:** `Free` (or use student credits for `Starter`)

5. **Add Environment Variables:**
   Add the following variables in the **Environment** tab:
   - `PORT` = `10000` (Render defaults to 10000)
   - `NODE_ENV` = `production`
   - `FRONTEND_URL` = `https://your-frontend.vercel.app` (The URL generated by Vercel)
   - `SUPABASE_URL` = `https://your-project.supabase.co`
   - `SUPABASE_PUBLISHABLE_KEY` = `your-supabase-publishable-key`
   - `SUPABASE_SERVICE_ROLE_KEY` = `your-supabase-service-role-key` (Must keep secret)
   - `GEMINI_API_KEY` = `your-google-gemini-key`
   - `SMTP_HOST` = `smtp.gmail.com`
   - `SMTP_PORT` = `587`
   - `SMTP_USER` = `your-smtp-email`
   - `SMTP_PASS` = `your-smtp-app-password`
   - `SMTP_FROM` = `DevPulse <no-reply@yourdomain.com>`
   - `RAZORPAY_KEY_ID` = `your-razorpay-key`
   - `RAZORPAY_KEY_SECRET` = `your-razorpay-secret`

6. **Deploy:** Render will build and deploy the Express server.
   > [!IMPORTANT]
   > Render's **Free Tier** web services spin down (sleep) after 15 minutes of inactivity. When a new request arrives, it can take 30–50 seconds to spin back up.
   > **Workaround for Students:**
   > 1. Use the Render student credits to upgrade the Web Service to the **Starter** tier ($7/month), which keeps it alive 24/7.
   > 2. Alternatively, set up a free cron job on [Cron-Job.org](https://cron-job.org) to ping your backend's `/api/health` endpoint every 14 minutes, keeping it awake for free!

## 3. Advanced Backend Hosting: AWS Free Tier & Student Academy

AWS offers the most powerful infrastructure, but it has a steeper learning curve. As a student, you have two primary entry points:
- **AWS Free Tier:** 12 Months of free access to select services.
- **AWS Academy / AWS Educate:** Free labs and starter credits through your institution.

### The AWS Free Services We Can Use:
1. **EC2 (Elastic Compute Cloud):** 750 hours/month of a `t2.micro` or `t3.micro` instance (Linux/Windows) for 12 months. This is perfect for running the Express server.
2. **RDS (Relational Database Service):** 750 hours/month of a single `db.t2.micro` or `db.t3.micro` database for 12 months (alternative to Supabase).

### Step-by-Step EC2 Node.js Deployment (Zero Cost)

#### Step 3.1: Launch the Instance
1. Sign up/Log into the [AWS Console](https://aws.amazon.com).
2. Go to **EC2 Dashboard** → **Launch Instance**.
3. **Configure the Instance:**
   - **Name:** `DevPulse-Backend`
   - **OS (AMI):** Ubuntu Server (Choose the **Free Tier Eligible** version).
   - **Instance Type:** `t2.micro` (or `t3.micro` depending on region eligibility).
   - **Key Pair:** Create a new key pair (`.pem` file), save it securely. You will need this to SSH into the server.

#### Step 3.2: Security Group Configuration (Firewall)
Under **Network Settings**, configure the Security Group rules:
- Allow **SSH** (Port 22) from your IP only (security best practice).
- Allow **HTTP** (Port 80) from anywhere (`0.0.0.0/0`).
- Allow **HTTPS** (Port 443) from anywhere (`0.0.0.0/0`).
- Allow **Custom TCP** (Port 5000) if you want to bypass a reverse proxy (Not recommended, use Nginx reverse proxy instead).

Launch the instance.

#### Step 3.3: Connect and Setup the Server
Open your terminal (or Git Bash) and SSH into the instance:
```bash
chmod 400 your-key.pem
ssh -i "your-key.pem" ubuntu@your-ec2-public-dns
```

Once inside Ubuntu, run the setup commands:
```bash
# 1. Update OS packages
sudo apt update && sudo apt upgrade -y

# 2. Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 3. Verify installation
node -v && npm -v

# 4. Clone your repository
git clone https://github.com/akshayrajput12/devpulse.git
cd devpulse/backend

# 5. Build the application
npm install
npm run build
```

#### Step 3.4: Configure PM2 (Process Manager)
To ensure the backend runs continuously in the background and restarts automatically on crash or server reboot:
```bash
sudo npm install -g pm2

# Start the Express server
pm2 start dist/server.js --name "devpulse-backend"

# Set up auto-start on reboot
pm2 startup systemd
# Copy-paste the command generated by the output of the line above
pm2 save
```

#### Step 3.5: Set up Nginx as a Reverse Proxy (Port 80 → Port 5000)
To avoid having to type `:5000` in your backend URLs and to enable easy SSL setup:
```bash
# Install Nginx
sudo apt install nginx -y

# Configure Nginx config
sudo nano /etc/nginx/sites-available/default
```

Replace the file contents inside `location /` block:
```nginx
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    server_name yourdomain.com; # Add your custom namecheap domain here

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```
Save the file (`Ctrl+O`, `Enter`, `Ctrl+X`) and restart Nginx:
```bash
sudo systemctl restart nginx
```

#### Step 3.6: Configure Free SSL (HTTPS) with Let's Encrypt
```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d yourdomain.com
```
Follow the prompts. Certbot will automatically rewrite the Nginx configuration to support HTTPS, configure auto-renewal of the SSL certificates, and redirect all HTTP traffic to HTTPS securely.

## 4. Supporting Database & Services (100% Free)

To keep operating costs at **absolute zero**, configure these free ecosystem databases:

### Database: Supabase Free Tier
- **What you get:** 1 Free PostgreSQL database, 500MB storage, 50,000 monthly active authentication users.
- **Setup:** Create a free project, configure the tables using the [database schema listing in the README.md](file:///d:/Devpulse/code-pulse/README.md#16-database-schema-all-tables), and plug the project credentials directly into your backend's environment variables.

### Cache & Rate Limiting: Upstash Serverless Redis
- **What you get:** 1 Free serverless Redis database with **10,000 free commands per day**.
- **Setup:** Log into [Upstash](https://upstash.com), create a free Redis database, and plug in the credentials. This fully covers rate-limiting and similarity caching for small-to-medium development workspaces.

### SMTP Transactional Email: Resend Free Tier
- **What you get:** 3,000 emails/month on a free tier (100 emails/day), custom domain support.
- **Setup:** Log into [Resend](https://resend.com), add your Namecheap student domain, verify the DNS records, generate an API key, and configure it under your backend SMTP settings.
